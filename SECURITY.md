# Security Policy

## Supported Versions

ContextR follows semantic versioning. Security fixes are provided for the latest release.

## Reporting a Vulnerability

If you discover a security vulnerability, we appreciate your help in disclosing it responsibly.

### How to report

- **Primary method:** Use the "Report a vulnerability" button on the Security tab of the GitHub repository. This uses GitHub Security Advisories to report privately. Only report vulnerabilities using this method so the issue can be addressed before public disclosure.
- **Fallback:** If the vulnerability reporting option is unavailable, open a regular GitHub issue. Do **not** include exploit details or sensitive information in public issues.

### Response timeline

The maintainer will acknowledge your report within 5 business days.
