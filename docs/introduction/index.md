This section defines the operational scope of ContextR and provides the recommended onboarding path for production systems.

## Decision Guidance

- Use ContextR when context must flow across transport and async boundaries with deterministic behavior.
- Avoid ContextR for identity and authorization concerns that belong to auth infrastructure.
- Prefer explicit contracts and policies over ad-hoc header handling.
